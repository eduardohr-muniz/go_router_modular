# Navegação

É `go_router` por baixo, então os métodos padrão de navegação no `BuildContext`
funcionam como você espera. Além disso, o `go_router_modular` adiciona variantes
**assíncronas** que aguardam as dependências do módulo de destino terminarem de
ser registradas.

## Navegação padrão

```dart
context.go('/home');                 // substitui a pilha
context.push('/details');            // empilha por cima
context.pop();                       // remove a rota atual
context.replace('/settings');        // substitui a rota atual

context.goNamed(
  'user',
  pathParameters: {'id': '42'},
  queryParameters: {'tab': 'posts'},
);
```

## Navegação assíncrona

As variantes assíncronas retornam um `Future` que completa apenas depois que os
binds do módulo de destino foram registrados e a nova página foi construída. Cada
uma recebe um payload `extra` opcional e um callback `onComplete`.

```dart
await context.goAsync('/dashboard');

await context.goNamedAsync(
  'user',
  pathParameters: {'id': '42'},
  queryParameters: {'tab': 'posts'},
  onComplete: () => print('dependências prontas'),
);
```

Os mesmos formatos existem para cada verbo de navegação:

- `goAsync` / `goNamedAsync`
- `pushAsync` / `pushNamedAsync`
- `pushReplacementAsync` / `pushReplacementNamedAsync`
- `replaceAsync` / `replaceNamedAsync`

Assinaturas:

```dart
Future<void> goAsync(String route, {Object? extra, VoidCallback? onComplete});

Future<void> goNamedAsync(
  String name, {
  Map<String, String> pathParameters = const {},
  Map<String, String> queryParameters = const {},
  Object? extra,
  VoidCallback? onComplete,
});
```

<Callout type="info">
  Prefira `goAsync` em vez de `go` quando precisar rodar código logo após as
  dependências do destino estarem prontas — por exemplo, ler um controller que o
  módulo de destino registra, ou aguardar a navegação em um fluxo. Para navegação
  simples, `go` resolve.
</Callout>

## Voltando até uma localização

```dart
context.popUntil('/home');         // remove até esta localização ficar no topo
context.popUntilNamed('home');     // remove até esta rota nomeada ficar no topo
```

## Lendo o estado da rota

Use a fachada `GoRouterModular` para ler o estado da rota atual a partir de um
`BuildContext`. Os métodos seguem a convenção `...Of(context)`:

```dart
final state    = GoRouterModular.routerStateOf(context);        // o GoRouterState
final path     = GoRouterModular.currentPathOf(context);        // o caminho atual
final id       = GoRouterModular.pathParamOf(context, 'id');    // um parâmetro de caminho
final params   = GoRouterModular.pathParamsOf(context);         // todos os parâmetros de caminho
final ref      = GoRouterModular.queryParamOf(context, 'ref');  // um query parameter
final queries  = GoRouterModular.queryParamsOf(context);        // todos os query parameters
final uri      = GoRouterModular.currentUriOf(context);         // o Uri atual
final location = GoRouterModular.currentLocationOf(context);    // a location atual
final payload  = GoRouterModular.extraOf<MyPayload>(context);   // o extra tipado
```

<Callout type="info">
  Os atalhos por extension `context.getPathParam('id')`, `context.getPath` e
  `context.state` continuam funcionando, mas estão **depreciados** em favor de
  `GoRouterModular.pathParamOf`, `currentPathOf` e `routerStateOf`. Serão
  removidos em uma major futura.
</Callout>

Se preferir, você também pode usar diretamente os utilitários do `go_router`
(`GoRouterState`, `context.go`, `context.push`, …), que são re-exportados pelo
barril do pacote — basta importar `package:go_router_modular/go_router_modular.dart`.

## Buscando dependências

Resolva uma dependência registrada a partir de um widget com `context.read<T>()`:

```dart
final controller = context.read<HomeController>();
```

## Navegando fora da árvore de widgets

`modularNavigatorKey` é uma `GlobalKey<NavigatorState>` que te dá o context do
navigator atual em qualquer lugar — services, interceptors, callbacks em segundo
plano.

```dart
final context = modularNavigatorKey.currentContext;
context?.go('/login');
```

## Na web

Deep links, a barra de URL e o voltar/avançar do navegador funcionam — é
`go_router` por baixo, então parâmetros de caminho e de query trafegam pela URL.
