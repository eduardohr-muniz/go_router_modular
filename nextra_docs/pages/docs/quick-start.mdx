# Quick Start

Set up a modular Flutter app in three steps.

## 1. Install

```bash
flutter pub add go_router_modular
```

## 2. Configure

```dart filename="lib/main.dart"
void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await Modular.configure(
    appModule: AppModule(),
    initialRoute: '/',
    debugLogDiagnostics: true,        // Log module injection events
    debugLogDiagnosticsGoRouter: true, // Log GoRouter navigation events
    debugLogEventBus: true,            // Log EventModule events
  );
  runApp(AppWidget());
}
```

> Set all `debug*` flags to `false` in production builds.

```dart filename="lib/src/app_widget.dart"
class AppWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context) {
    return ModularApp.router(title: 'My App');
  }
}
```

### Overriding router options

`ModularApp.router` wires the modular router for you. If you need a router option
it doesn't expose — for example `observers`, `redirect`, or `refreshListenable` —
use `MaterialApp.router` directly and `copyWith` on `Modular.routerConfig`.
Everything from `configure` is reused; you override only what you list.

```dart filename="lib/src/app_widget.dart"
class AppWidget extends StatelessWidget {
  const AppWidget({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp.router(
      // Reuses the whole modular configuration, overriding only `observers`.
      routerConfig: Modular.routerConfig.copyWith(
        observers: [MyAnalyticsObserver()],
      ),
      builder: (context, child) => ModularLoader.builder(context, child),
    );
  }
}
```

You can override any router option this way — `observers`, `redirect`,
`refreshListenable`, `errorBuilder`, `errorPageBuilder`, `redirectLimit`,
`navigatorKey`, `restorationScopeId`, and more.

> The derived router is memoized: the first `copyWith` call builds it and later
> rebuilds return the same instance, so your navigation state is preserved.

## 3. Create Modules

```dart filename="lib/src/app_module.dart"
class AppModule extends Module {
  @override
  List<ModularRoute> get routes => [
    ModuleRoute('/', module: HomeModule()),
  ];
}
```

```dart filename="lib/src/modules/home/home_module.dart"
class HomeModule extends Module {
  @override
  FutureBinds binds(Injector i) { // Optional
    i.addSingleton<HomeController>((i) => HomeController());
  }

  @override
  List<ModularRoute> get routes => [
    ChildRoute('/', child: (context, state) => HomePage()),
  ];
}
```

That's it. Your modular app is ready.

## Project Structure

A typical modular project looks like this:

```
lib/
  main.dart
  src/
    app_module.dart
    app_widget.dart
    modules/
      home/
        home_module.dart
        pages/
          home_page.dart
      products/
        products_module.dart
        pages/
          products_page.dart
```

Each module is a self-contained unit with its own routes, dependencies, and pages.
